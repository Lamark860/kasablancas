# Брифинг — продукт-менеджеру и дизайнеру

**Когда:** 2026-05-18
**Что обсуждать:** разворот продукта в воронку лидов, что собрано в MVP, что подумать вам — чтобы поднять конверсию, удержание и монетизацию.

> Скриншоты текущего состояния — `design/screenshots/01-anketa-empty.png` (форма), `design/screenshots/02-result-with-lead.png` (страница результата с раскрытой формой консультации).

## 1. Что изменилось

До этой итерации проект был **внутренним инструментом эксперта**: эксперт сам заполнял анкету гостьи, сам выбирал главное дерево и сопровождающие, отдавал PDF. Сейчас весь этот флоу остаётся «back-office» (`/registry`, `/intake`, `/expert/...`, `/leaf/{token}` — без изменений).

Что появилось — **публичная воронка лидов на главной**:

1. Любой человек заходит на `/` и **сам** заполняет короткую анкету: имя, дата рождения, цвет глаз, место рождения (с геокодером — нужно для натальной части подбора).
2. На той же странице мгновенно получает **авто-результат**: главное тотемное дерево + до 5 сопровождающих растений + причины подбора от сработавших оракулов (друиды, зодиак, цвет глаз, нумерология имени, натальная карта если есть место).
3. Снизу — чекбокс **«хочу подробную консультацию»**. Клик → форма с полем телеграм/телефон. Сабмит → захват лида в админку.
4. Эксперт работает с лидами в `/admin/lead/list`: статусы (`new → contacted → won/lost`), заметки, экспорт CSV/JSON.

Бренд на публичных страницах пока **«Hortus Animæ»** (тот же латинизм, что был внутри). Это плейсхолдер — переименование на повестке, см. §4.

## 2. Новая воронка по шагам

```
┌───────────────┐    ┌───────────────┐    ┌──────────────────┐    ┌─────────────┐
│  Главная /    │    │  Результат    │    │  Чекбокс         │    │  Лид в БД   │
│  Анкета       │───▶│  Дерево +     │───▶│  «хочу           │───▶│  Эксперт    │
│  4 поля       │    │  сопровожд.   │    │  консультацию»   │    │  звонит     │
└───────────────┘    └───────────────┘    └──────────────────┘    └─────────────┘
   2-3 минуты         мгновенно            опциональный шаг       вручную сейчас
```

**Что собираем:** имя, ДР, цвет глаз, место рождения, телеграм/телефон (только если поставили галку), город (выделяется из места рождения, для аналитики).

**Что НЕ собираем (сознательно):** пол, цвет волос, отчество/фамилия, садовые поля (зона/почва/свет — это «технические» поля эксперта, клиентка в них не разбирается). Если PM решит, что что-то надо добавить — обсудим.

**Платных шагов в воронке пока нет.** Чекбокс — бесплатный, эксперт звонит и продаёт консультацию вручную (15 мин / 500 ₽, 30 мин / 1000 ₽, час / 2000 ₽, выезд — это пакеты, согласованные с владельцем продукта; см. `BACKLOG.md`).

## 3. Что уже работает в коде

Прокликать локально:
- `http://localhost:3100/` — анкета и результат.
- `http://localhost:3100/registry` — back-office эксперта (старый реестр).
- `http://localhost:8100/admin/lead/list` — таблица лидов.

API:
- `POST /leads/` — захват, сохраняет анкету + snapshot результата (`main_plant_slug/name` + `companions` JSON) + город + (опционально) referrer.
- `GET /leads/stats` — счётчики: всего, с галкой консультации, по статусу, топ городов.

Что **в тех. долге** (см. `BACKLOG.md` в корне):

- **Мобильная вёрстка** — основной канал, ничего не свёрстано под телефон. Только desktop-first сейчас.
- **Имя продукта** — открыто.
- **Тексты-якори** на растения — сейчас на странице результата показываются существующие `short_story` (одинаковые для всех, кто получил это дерево). В отложке — отдельная таблица `plant_texts (plant_id, oracle_system, snippet)` и LLM-перефразирование через router.ai, чтобы текст звучал «под клиентку», а не как штамп.
- **Фазы луны / окна посадки** — для каждого дерева, выводится на странице результата отдельным блоком.
- **Платные тиры** — выбор пакета прямо в форме лида, а не вручную по телефону.
- **Расширенный план за ~100 ₽** с приватными деталями (размер дерева, что закопать рядом, прядь волос).
- **Платёжка**, **VPS**, **аналитика-визуализация**.

## 4. На что хотим ваш взгляд

### Продукт-менеджеру — про продажи и монетизацию

1. **Имя продукта.** «Hortus Animæ» — латинизм, на холодную аудиторию не считывается, может стать барьером. Варианты обсуждались (Своё дерево / Древесница / Берегиня / Корни), не закрыты. **Вопрос:** какие ассоциации хотим бренд оставить (мистика / тёплая ботаника / женственная сила / научно-сакральное) — это нужно зафиксировать до того как дизайнер начнёт переделывать визуал.

2. **Воронка после захвата лида.** Сейчас «эксперт звонит → продаёт пакет». Это упирается в время эксперта. **Вопрос:** какой минимально интересный пакет, чтобы конверсия из звонка в оплату была максимальной? И когда подключаем платёжку (СБП/ЮKassa) — стартуем с одного пакета или сразу с лесенки?

3. **Расширенный план за ~100 ₽ как первый платный шаг.** Не дороже импульсивной покупки кофе, даёт клиентке «глубину» (что закопать, фазы луны под её регион, размер дерева). **Вопрос:** показывать ли прямо на странице результата (сразу после галки консультации, чтобы перехватить тех, кто не доверится звонку), или только после бесплатного звонка?

4. **Sharing / вирусность.** Страница результата визуально красивая — может работать как «покажу подруге». **Вопрос:** нужна ли кнопка «поделиться своим деревом» (даёт sharable-ссылку с подбором, и подруга в один клик заходит к нам), и насколько это приоритет на этой стадии?

5. **Подписочная модель.** Возможна как «фазы луны для вашего дерева» / «месячные посевные окна по натальной карте» — рекуррентный продукт. **Вопрос:** идём в подписку, или фокус на разовые консультации + апсейл выезда?

6. **A/B-тесты.** Какие гипотезы у вас есть на первой ступени (заголовок / порядок полей / визуал главного дерева / цена расширенного плана)? Это влияет на то, какую инфраструктуру для экспериментов закладывать.

### Дизайнеру — про UX и впечатление

1. **Тон визуала.** Сейчас «викторианско-эзотерический» (UnifrakturCook display, бумажная фактура, винные акценты). Для платной аудитории это или отстройка («не как все»), или барьер («слишком вычурно»). **Вопрос:** оставляем направление и докручиваем, или перебираем регистр? Решение завязано на §4.1 (имя).

2. **Главное дерево на странице результата.** Сейчас — буква имени растения в круге. **Вопрос:** не нужна ли акварель / силуэт / иллюстрация дерева? Сильно повлияет на «вау-эффект» и желание поделиться.

3. **Мобильная вёрстка.** Главный канал, ничего не свёрстано. **Что нужно от вас:** мобильные макеты для двух ключевых экранов — анкета и страница результата (включая раскрытое поле контакта).

4. **Hero страницы.** Сейчас просто заголовок + форма. **Вопрос:** нужна ли над формой инфо-секция «как это работает» (3 шага, иллюстрации) — для холодного трафика повышает trust и закрывает «а это вообще что?».

5. **Социальный пруф.** Отзывы клиенток, фото настоящих посадок, кейсы. **Вопрос:** где в потоке размещать (на главной до формы / на странице результата над чекбоксом — чтобы дожать на консультацию)? Нужен сбор контента — обсудить с владельцем.

6. **Анкета в один экран vs квиз пошагово.** Сейчас все 4 поля сразу. У квиз-формата (тильда-стиль, «1/4 → 2/4 → ...») часто выше completion-rate из-за sunk-cost. **Вопрос:** тестируем формат, или оставляем плоско?

7. **После сабмита лида.** Сейчас «✦ принято — мы скоро вам напишем». **Вопрос:** что добавить (ожидаемое время отклика, ссылка на телеграм/инсту бренда для подписки, превью следующего шага)? Это последняя точка касания на этой сессии — не теряем её.

8. **Геокодер.** Поле «место рождения» с автокомплитом и выбором между кандидатами (если найдено больше одного — например, 2 разные «Москвы»). UX скопирован из экспертского интейка. **Вопрос:** для клиента это не слишком технично? Можем упростить (например, только город без района), но потеряем точность асцендента в натальной карте.
