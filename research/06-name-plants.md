# Связи "имя → растение"

Самый "магический" слой для клиента: услышать, что её **имя** связано с конкретным растением — это сильное эмоциональное попадание.

## Подходы

### A. Прямая этимология / "именное" растение

Имена, которые буквально означают растение или его качество:

| Имя | Растение |
|-----|----------|
| Лилия | Лилия |
| Роза, Розалия | Роза |
| Маргарита | Маргаритка |
| Виолетта | Фиалка |
| Дафна | Лавр |
| Ива | Ива |
| Ясмин(а) | Жасмин |
| Камилла | Ромашка (camomile) |
| Ирис, Ирина (через греч. Iris) | Ирис |
| Сюзанна | Лилия (евр. "лилия") |
| Фиона | "Виноградная лоза" / лес |
| Дарина / Дарья | "сильная" → дуб, ясень |
| Ева | "жизнь" → яблоня (древо познания), любые "райские" |
| Анастасия | Бессмертник (греч. "воскрешение") |
| Александра | Лавр (победа) |
| Полина | Аполлон → лавр |
| Татьяна | Тысячелистник (по созвучию + связь с Татианой-мученицей) |

→ Список расширяемый. Можно вести прямую таблицу `имя → 1-3 растения` и наращивать её по факту запросов.

### B. Святцы (для православных имён)

Имя → день памяти святого → растения, цветущие в этот день / атрибутируемые святому.

Примеры:
- Иван (день Ивана Купалы 7 июля) → иван-чай, папоротник, иван-да-марья
- Илья (Илья-пророк, 2 авг.) → дуб (молнии), хлеба
- Николай (зимний, 19 дек.) → ель, омела
- Мария (Богородица) → роза, лилия, мирт
- Параскева → лён (покровительница ткачества)

Полная таблица собирается через православный календарь имён (минея). Это работа на этап 2.

### C. Цветочно-травянистая семантика имени

Если ни А ни Б не сработали — берём **значение имени** и подбираем растение по смыслу:
- "светлая" → берёза, ромашка, белые цветы
- "благородная" → пион, роза, дуб
- "храбрая" → дуб, барбарис (шипы), чертополох
- "тихая/скромная" → ландыш, фиалка, мох
- "огненная" → красные/жёлтые цветы, рябина
- "морская" → ирис, кувшинка

Это уже работа эксперта, не таблица. Но в инструменте можно держать справочник "значение → растение" и подсказывать.

### D. Нумерология имени

Сумма букв (по табличке а=1, б=2, ...) → одно число → растение по числу. Параллельно с нумерологией даты рождения. Если оба числа дают одно растение — сильный знак.

## В каком виде хранить в БД

```
names: [
  { name: "Татьяна", plants: ["тысячелистник", "татарник", "вереск"], 
    via: "созвучие + святая Татиана", weight: 1.0 },
  { name: "Иван",    plants: ["иван-чай", "иван-да-марья", "папоротник"], 
    via: "Иван Купала (7 июля)", weight: 1.0 },
  ...
]
```

`via` нужно для нас — чтобы знать, какую "историю" рассказать клиенту, если выпало это растение.

## Как добавлять новые имена

Каждый клиент = повод добавить его имя в базу с растением(ями). Если пересечение получилось красивым — записываем "канон". Если нет — отмечаем как "пробел" и докручиваем со временем.
