# Roadmap — поэтапный план разработки

Документ для того, чтобы знать **что делать в каком порядке** и не утонуть. Каждый этап даёт **рабочее состояние**, которое можно показать коллеге и обсудить.

Ориентировочные оценки времени даны для одного разработчика, работающего по вечерам/выходным.

---

## Этап 0 — Подготовка (1-2 вечера)

**Цель:** окружение готово, проект собирается, "Hello World" в браузере.

- [ ] Установить Python 3.12, Node 20, uv, pnpm, Docker Desktop
- [ ] Создать монорепо по структуре из `01-project-structure.md`
- [ ] `api/`: инициализировать через `uv init`, добавить FastAPI, uvicorn, ruff
- [ ] `web/`: инициализировать через `pnpm create nuxt`, базовая страница
- [ ] Написать минимальный `docker-compose.yml` (api + web)
- [ ] Проверить: `docker compose up` → открывается `localhost:3000` (Nuxt) и `localhost:8000/docs` (FastAPI Swagger)
- [ ] Закоммитить в git, подключить репо (GitHub/Gitea)

**Чек:** оба сервиса стартуют, между ними сделан тестовый запрос (Nuxt дёргает `/api/ping`, получает `{ok: true}`).

## Этап 1 — БД и модели (2-3 вечера)

**Цель:** схема БД накачена, есть пустая админка.

- [ ] Добавить SQLAlchemy 2.0, Alembic, sqladmin, pydantic-settings
- [ ] Написать ORM-модели в `models/` по `02-database-schema.sql`:
  - Person, Plant, Oracle, OracleEntry, Recommendation, NatalChart
- [ ] Настроить Alembic, сгенерировать первую миграцию `alembic revision --autogenerate -m "init"`
- [ ] Прогнать `alembic upgrade head` → SQLite-файл создан
- [ ] Подключить sqladmin на `/admin`, зарегистрировать все модели
- [ ] Проверить: открывается `localhost:8000/admin`, можно создать тестового Person и тестовое Plant руками

**Чек:** через админку добавил клиента и 2-3 растения, всё сохранилось в БД.

## Принципы по ходу разработки

1. **Каждый этап — рабочее состояние.** В конце этапа проект должен запускаться и делать что-то полезное. Не накатываешь 5 этапов сразу.

2. **Один оракул за раз.** Не пытайся написать сразу 8. Сделай первый идеально (с тестами), потом следующий по образцу.

3. **Сначала бэк, потом фронт.** Когда API готов — фронт пишется быстро. Обратное неверно.

4. **Reseed безболезненно.** Команда `python -m vlad.seed` должна **очищать и заново заливать** все эзотерические данные (oracle_entries, plants справочник). Person и Recommendation — не трогать. Это даёт право переписывать JSON-сиды без боли.

5. **Не оптимизируй преждевременно.** SQLite справится с десятками тысяч растений и сотнями клиентов. Postgres понадобится только когда выйдешь в паблик.

6. **Не отвлекайся на дизайн админки.** sqladmin "уродливый" — это нормально, это инструмент эксперта, не клиента. Красоту делаешь только для клиент-режима и PDF.
