# Handoff — пакет для разработки

Этот раздел — для того, кто будет писать код (в данном случае — ты сам).

## Что здесь

| Файл | Что внутри |
|------|------------|
| `00-stack.md` | Рекомендуемый стек технологий с обоснованием и альтернативами |
| `01-project-structure.md` | Структура папок и модулей |
| `02-database-schema.sql` | Готовая SQL-схема БД (SQLite/Postgres) |
| `03-oracle-interface.md` | Контракт оракула + полный пример реализации |
| `04-roadmap.md` | Поэтапный план разработки (что делать в каком порядке) |
| `05-natal-chart.md` | Как считать натальную карту (Swiss Ephemeris) |

## Порядок чтения

1. **`04-roadmap.md`** — посмотри этапы целиком, чтобы понимать масштаб
2. **`00-stack.md`** — определись со стеком прежде чем что-либо писать
3. **`01-project-structure.md`** — заведи структуру папок
4. **`02-database-schema.sql`** — накати схему в SQLite (это 5 минут)
5. **`03-oracle-interface.md`** — реализуй один оракул (друид-деревья — самый простой), убедись что работает
6. Далее — по `04-roadmap.md`

## Важно

В корне проекта есть документация **доменной части** (`research/00-overview.md` ... `11-feng-shui-vastu.md`). Это про то **что считать**. Handoff — про то **как реализовать**. Читать в связке.

Дизайн-прототип — в `Prototype.html`. Открой в браузере, потыкай — это эталон UX, к которому нужно прийти. Архитектура поддерживает именно эти экраны.
