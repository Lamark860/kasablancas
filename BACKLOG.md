# Бэклог / тех. долг

Здесь собираем всё, что отложено: задачи, до которых дойдут руки после текущего этапа разворота продукта (внутренний инструмент → публичная воронка лидов). Каждая запись — короткий блок с заголовком, причиной откладывания и условием выполнения.

---

## UX / фронт

### Мобильная вёрстка
- **Откуда:** пользователь, 2026-05-18.
- **Что:** все клиентские страницы (`/`, страница результата, `/leaf/{token}`) должны быть рабочими и красивыми на телефоне — основной канал, лиды будут приходить с мобильных. Сейчас вёрстка десктоп-first, на узких экранах не проверялась.
- **Когда:** после того как продукт-воронка собран в новом формате (задачи A–D готовы) и согласован с продакт-менеджером и дизайнером. То есть **сначала** контент/поток на десктопе → потом адаптив с дизайнером.
- **Что не сюда:** интерьерные/админ-экраны (`/registry`, `/intake`, `/expert`, `/admin`) — они для эксперта, десктопа достаточно.

### Имя продукта
- **Откуда:** разворот 2026-05-18.
- **Что:** на публичных страницах сейчас «Hortus Animæ» — латинизм с внутренней эпохи. Для клиентской воронки нужно русское/понятное имя, которое не отпугнёт. Варианты обсуждались, выбор отложен.
- **Когда:** перед открытием публичного доступа (когда поедем на VPS) или раньше, если придёт идея.

## Подбор / контент

### Тексты-якори по растениям × оракулам
- **Что:** отдельная таблица `plant_texts (plant_id, oracle_system, snippet)` + поле `Plant.client_intro`. Сейчас на странице результата показываются существующие `plant.short_story` — это «штамп», одинаков для всех. Якори дают «конструктор из причин» (текст склеивается из тех систем, что реально сработали для конкретной клиентки) и потом — основу для LLM-перефразирования.
- **Когда:** после первой ступени воронки. Эксперт должен заполнить кусочки в админке — это отдельная задача на её время.

### LLM-перефразирование описаний (router.ai / openrouter)
- **Что:** на странице результата текст растения собирается из якорей и пропускается через LLM, чтобы звучать «под клиентку» (имя, причины). У пользователя есть аккаунт router.ai.
- **Когда:** после того как таблица якорей заполнена. Раньше — нечего перефразировать.

### Фазы луны / окна посадки
- **Что:** для каждого дерева — рекомендованные окна посадки по фазе луны. На страницу результата выводится отдельным блоком.
- **Когда:** после первой ступени. Либо забить руками по каждому растению в админке, либо считать по астрономическому правилу — решим, когда дойдёт.

### Цвет волос как фактор подбора
- **Что:** новый оракул, который сопоставляет цвет волос растениям (или модификатор существующих оракулов). Само поле «цвет волос» в публичной анкете сейчас не нужно (см. разворот 2026-05-18) — будет, когда эксперт пропишет правило.
- **Когда:** после того как эксперт сформулирует логику соответствия.

## Монетизация

### Платные тиры консультаций
- **Что:** на странице результата чекбокс «хочу консультацию» сейчас просто захватывает лида (эксперт перезванивает). Следующая ступень — выбор пакета (15 мин / 500 ₽, 30 мин / 1000 ₽, час / 2000 ₽, выезд) прямо в форме лида.
- **Когда:** после того как поток первых лидов покажет, какой пакет реально берут. Сначала собираем данные, потом структурируем.

### Расширенный план за ~100 ₽ с приватными деталями
- **Что:** платный апсейл сразу после результата: размер дерева, что закопать рядом (прядь волос и т.п.), фазы луны под конкретный географический регион клиента. Доступно только клиенту после оплаты.
- **Когда:** после того как платные консультации заработают и подключена платёжка.

### Платёжная интеграция
- **Что:** приём оплат (СБП / ЮKassa / что-то ещё). Без неё все платные тиры — телефонные.
- **Когда:** после первого валидированного спроса (несколько проданных консультаций через переписку).

## Инфраструктура

### Хостинг на VPS
- **Что:** сейчас стек живёт локально в Docker Desktop. Для публичной воронки нужен домен и VPS (DNS, HTTPS, обновления).
- **Когда:** после первого рабочего прототипа воронки (задачи A–D + дизайн с дизайнером).

### Аналитика лидов
- **Что:** на странице админки «Лиды» (задача D) сейчас базовые цифры (всего, с галкой, по городам). Полная визуализация (графики, конверсия по дням, источники-UTM) — отдельной задачей.
- **Когда:** когда наберётся хотя бы пара сотен лидов и появится что визуализировать.
