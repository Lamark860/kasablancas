  HORTUS ANIMÆ — как запустить проект на своём компьютере
═══════════════════════════════════════════════════════════════════

Это прототип помощника эксперта по подбору растений.
Чтобы посмотреть и покликать у себя на ноуте — три шага.


───────────────────────────────────────────────────────────────────
  ШАГ 1.  ПОСТАВИТЬ DOCKER DESKTOP
───────────────────────────────────────────────────────────────────

Это бесплатная программа, которая нужна, чтобы запустить проект
не разбираясь в нутрянке.

  • Скачай отсюда:
        https://www.docker.com/products/docker-desktop/

    На странице выбери свою операционную систему:
        — Mac на Apple Silicon (M1/M2/M3)
        — Mac на Intel
        — Windows

  • Запусти скачанный файл и пройди установку (всё «Далее → Далее»,
    ни на что не соглашайся отдельно — пусть ставит как предлагает).

  • После установки запусти программу «Docker Desktop». На Mac
    в верхней панели появится значок кита. На Windows — в трее
    (правый нижний угол) появится тот же кит. Подожди минуту, пока
    кит перестанет анимироваться — это значит Docker готов.

    ВАЖНО: при первом запуске Docker Desktop может попросить
    создать учётку — это можно пропустить (кнопка «Skip» или
    «Continue without signing in»).


───────────────────────────────────────────────────────────────────
  ШАГ 2.  РАСПАКОВАТЬ АРХИВ
───────────────────────────────────────────────────────────────────

Распакуй полученный архив в любую удобную папку (например,
прямо на Рабочий стол). Не нужно ничего «устанавливать» — просто
обычная разархивация. Внутри папки увидишь файлы, в том числе
этот, который ты сейчас читаешь.


───────────────────────────────────────────────────────────────────
  ШАГ 3.  ЗАПУСТИТЬ
───────────────────────────────────────────────────────────────────

  ▸ На Mac:
        двойной клик по файлу    start.command

    Если Mac ругается «нельзя открыть, потому что разработчик не
    подтверждён» — нажми правой кнопкой на start.command, выбери
    «Открыть», в появившемся диалоге снова «Открыть». Потом этот
    файл будет запускаться обычным двойным кликом.

  ▸ На Windows:
        двойной клик по файлу    start.bat


Откроется чёрное окно — не пугайся, это нормально, оно само
сообщит, что делает. ПЕРВЫЙ запуск идёт 5–10 минут (скачивает
и собирает всё, что нужно). Все следующие — несколько секунд.

Когда всё готово — окно само откроет браузер по адресу
http://localhost:3100/  Это и есть наш интерфейс.

Чёрное окно можно закрыть, на работу проекта это не повлияет.


───────────────────────────────────────────────────────────────────
  ЧТО МОЖНО ПОТЫКАТЬ
───────────────────────────────────────────────────────────────────

  1. На главной — список «гостий» (клиенток).
     Жми «+ новая гостья» наверху справа.

  2. В форме введи имя, дату рождения (важная штука — от неё
     зависит подбор), желательно цвет глаз и USDA-зону сада.
     Жми «✦ свести источники ✦».

  3. Откроется «экспертный лист» с пулом растений. У каждого —
     откуда оно пришло (друидский календарь / зодиак / цвет
     глаз / нумерология имени / имя), сколько раз пересеклось,
     и текстовая «причина» от каждой системы.

     Слева на карточках — чек-боксы «в избранное» и «главное».
     Отметь 3–5 растений, выбери одно главное, можешь оставить
     заметку. Жми «сохранить и открыть лист гостьи».

  4. Откроется «лист гостьи» — это уже формат, который видит
     клиентка: одно главное дерево, сопровождающие, без чисел
     и источников. Кнопка «✦ скачать pdf ✦» — даёт PDF-файл,
     который можно распечатать или отправить.

Чтобы посмотреть админку с сырыми данными (растения, оракулы,
правила) — http://localhost:8100/admin/  она «техническая»,
не для клиента.


───────────────────────────────────────────────────────────────────
  КАК ОСТАНОВИТЬ
───────────────────────────────────────────────────────────────────

Когда наклацался — остановить можно так:

  ▸ На Mac:    двойной клик по    stop.command
  ▸ Windows:   двойной клик по    stop.bat

Все введённые гостьи и сохранённые подборы остаются в проекте,
никуда не теряются — при следующем старте всё на месте.


───────────────────────────────────────────────────────────────────
  ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК
───────────────────────────────────────────────────────────────────

  • «Docker не установлен» / «Docker Desktop не запущен» —
    значит первый шаг не доделан. Открой приложение Docker
    Desktop, подожди пока кит-значок успокоится, попробуй
    запустить start.command/start.bat снова.

  • Браузер не открылся сам — открой его руками и зайди на
        http://localhost:3100/

  • Страница долго грузится после первого запуска — нормально,
    дай ещё минуту, контейнер прогревается.

  • Хочется «начать с чистого листа» (удалить всех тестовых
    клиенток) — останови проект через stop.command/stop.bat,
    удали папку проекта целиком, распакуй архив заново.

  • Что-то совсем не работает — пришли сюда скриншот чёрного
    окна, разберёмся.
