## Как читать эти материалы

1. Открой `Prototype.html` в браузере — это эталон UX
2. Прочти `research/00-overview.md` — общая концепция продукта
3. Просмотри `research/` — чтобы понимать, какие системы и данные у нас есть
4. Перейди в `handoff/README.md` — оттуда вход в инженерную документацию
5. По `handoff/04-roadmap.md` — иди этапами

## TL;DR что это за продукт

- **Этап 1 (сейчас):** внутренний инструмент для эксперта-ландшафтника. На входе — данные клиента (дата/место рождения, имя, цвет глаз). На выходе — пул из 10-15 растений-кандидатов с указанием, из каких "оракулов" они выпали и сколько раз.
- **Эксперт** видит источники и интерпретирует.
- **Клиент** видит готовый красивый отчёт без указания источников — это "авторская методика".
- **Этап 2 (потом):** публичный бот с автоотчётом → лид-генерация на консультацию.

## Стек (если коротко)

- **Frontend:** Nuxt 3 + Vue 3 + TypeScript
- **Backend:** FastAPI + SQLAlchemy 2 + Alembic
- **БД:** SQLite (этап 1) → PostgreSQL (этап 2)
- **Натальная карта:** pyswisseph
- **PDF:** WeasyPrint
- **Хостинг:** локально → VPS в РФ (Selectel/Timeweb)

Подробнее — в `handoff/00-stack.md`.
