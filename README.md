# Hortus Animæ (Vlad rev1)

Внутренний инструмент ландшафтного дизайнера для подбора растений клиенту на пересечении эзотерических и символических систем. На входе — данные гостьи (дата/место рождения, имя, цвет глаз, сад). На выходе — пул из 10–15 растений-кандидатов, главное дерево и красивый клиентский PDF.

> Это онбординг для коллеги. Если ты разработчик — после первого запуска иди в [`handoff/`](handoff/). Если эксперт-куратор — в [`ADMIN.md`](ADMIN.md). Если просто запустить и пощёлкать — продолжай отсюда.

---

## Что нужно поставить (один раз)

1. **Docker Desktop** — бесплатная программа, в которой будет работать весь проект. Никаких других зависимостей (Node, Python, БД) ставить руками не нужно — всё внутри Docker.
   - Скачать: https://www.docker.com/products/docker-desktop/
   - На странице выбери свою систему: Mac (Apple Silicon / Intel) или Windows.
   - Поставь как обычное приложение, на все вопросы — «Далее».
   - После установки **запусти** Docker Desktop. В трее (Win — правый низ, Mac — верх) появится значок кита. Подожди, пока кит перестанет двигаться — это значит Docker готов.

2. **Git** — чтобы скачать проект и потом получать обновления.
   - **Windows:** https://git-scm.com/download/win — поставь, на вопросах жми «Next».
   - **Mac:** скорее всего уже стоит. Открой `Terminal` и набери `git --version`. Если попросит установить — соглашайся.

> На Windows-е удобнее пользоваться **Git Bash** (ставится вместе с git): он умеет те же команды, что и Mac-овский Terminal.

## Первый запуск

Один раз скачать репозиторий и запустить. Дальше — двойной клик по `start`.

### 1. Скачать репозиторий

В терминале (Git Bash на Windows, Terminal на Mac):

```bash
cd <папка, где ты хранишь проекты>
git clone <ССЫЛКА-НА-РЕПО>
cd <название-папки-проекта>
```

Замени `<ССЫЛКА-НА-РЕПО>` на URL из адресной строки гитхаба — это что-то вроде `https://github.com/<юзер>/<репо>.git`.

### 2. Запустить

Открой папку проекта в **Проводнике** (Windows) или **Finder** (Mac) и сделай **двойной клик**:

- **Windows:** `start.bat`
- **Mac:** `start.command` (если Mac ругается «не удалось проверить разработчика» — правый клик → «Открыть»)

Откроется чёрное окно, в нём пробегут логи. **Первый раз это 5–10 минут** — Docker скачивает образы и собирает проект. Следующие запуски — секунды.

Когда увидишь строчки «Started vlad-api» и «Started vlad-web» — всё, можно открывать браузер.

### 3. Открыть в браузере

- **Главный UI** (реестр, intake, экспертная вкладка): http://localhost:3100/
- **Админка** (растения, оракулы, соответствия): http://localhost:8100/admin
- **Сырое API** (для проверки): http://localhost:8100/

## Остановить проект

Двойной клик по:
- **Windows:** `stop.bat`
- **Mac:** `stop.command`

Данные не теряются — они в `starter/api/data/local.db` и переживают остановку. Чтобы вообще удалить базу (вернуться к чистому состоянию) — удали этот файл (см. «Сбросить базу» ниже).

---

## Обновить проект до свежей версии

Когда я (или другой разработчик) внесли изменения и запушили — у тебя на машине нужно их подтянуть.

В терминале, **из папки проекта**:

```bash
git pull
```

Дальше:
- Если поменялся только фронт или контент — обычно ничего перезапускать не нужно, изменения подхватятся сами.
- Если что-то перестало работать — закрой проект (`stop.bat` / `stop.command`) и запусти снова (`start.bat` / `start.command`). При запуске Docker сам пересоберёт что нужно.
- Если в обновлении ушли правки каталога (растения, соответствия), они приедут как файлы в `starter/api/data/seed/` и подтянутся в БД при запуске **на чистой БД**. На существующей — посмотри пункт «Сбросить базу».

> Перед `git pull` стоит сначала запушить свои правки админки, если ты их успел сделать через «Выгрузить в JSON» (см. `ADMIN.md` раздел 7). Иначе git может конфликтовать.

## Сбросить базу (вернуться к чистому состоянию)

Иногда хочется начать с нуля — например, проверить, как поведёт себя проект на свежей машине, или вычистить тестовых гостий.

1. Останови проект (`stop.bat` / `stop.command`).
2. Удали `starter/api/data/local.db`.
3. Запусти заново. Из `starter/api/data/seed/*.json` поднимется каталог растений и оракулов, гостьи и подборы будут пустые.

## Если что-то сломалось

- **Не запускается, ругается «Docker не запущен»** → открой приложение Docker Desktop, дождись успокоившегося кита, запусти заново.
- **Запустилось, но http://localhost:3100/ показывает «не удаётся подключиться»** → подожди 30–60 секунд, фронт стартует чуть медленнее API. Если не помогло — открой Docker Desktop, посмотри что в «Containers»: `vlad-api` и `vlad-web` должны быть зелёные. Если красные — нажми на контейнер → «Logs», там видно причину.
- **PDF не открывается (Internal Server Error)** → скорее всего у гостьи не сохранён подбор. Зайди в `/expert/<id>`, отметь хотя бы одно растение в «electa» и нажми «отдать в лист». PDF появится.
- **Хочу попросить помощь** → перешли мне ссылку на гостью (например `http://localhost:3100/expert/16`) и скажи что не так. Если красное в Docker Desktop → скрин логов.

## Что не делать

- **Не коммить `starter/api/data/local.db`** — это твоя локальная БД, у каждого своя. Она в `.gitignore`.
- **Не коммить `.env`** — там могут быть локальные секреты.
- **Не пушить ветку напрямую в `main`**, если работаешь параллельно с кем-то ещё — лучше через Pull Request.
- **Не редактировать вручную таблицу `Рекомендации` в админке** — это история подборов, ломаются ссылки `/leaf/{token}`.
